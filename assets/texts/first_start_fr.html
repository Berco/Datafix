SVP. Lisez attentivement!

Cette application a été créée pour configurer le script init.d (Datafix). Elle crée aussi les fichiers move_cache.txt et skip_apps.txt que vous connaissez déjà. En addition, l'application est aussi capable de nettoyer le cache ou effacer les données de n'importe quelle application même lors de la présence de liens symboliques. Liens qui peuvent avoir été créée par ce script ou n'importe quel autre script.

move_cache.txt
Le fichier move_cache.txt est créé en cochant les cases sur l'écran principal. Les applications écrites dans ce fichier entreposeront les données des caches dans /data/data, permettant éventuellement de sauver beaucoup d'espace sur la rapide partition /datadata/*. Les applications Gmail et Download sont sélectionné par défaut.

skip_apps.txt
Les applications écrites dans ce fichiers verront TOUTE leur données entreposés dans la très grande mes aussi très lente partition /data sous /data/data/*.

Bouton Installer
Une fois ce bouton appuyé vous aurez une boite de confirmation, lisez attentivement ce qui sera ajouté à chaque fichier. Vous pouvez sois annuler, installer sans redémarrer ou installer et redémarré.
L'application décidera si le script doit être installé ou mis à jour. Une bonne pratique serait d'utilisé la fonction redémarrer avec backup nandroid. Le backup nandroid est ignoré si vous choisissez l'option d'installer sans redémarrer.

Détails d'application
Vous pouvez appuyer sur n'importe quelle application dans la liste. Ceci affichera l'emplacement actuel des données pour l'application et vous donne la possibilité d'effacer sont cache ou ses données. Les dossiers vide seront affiché à 4KB alors que les liens symbolique et les dossier inexistant seront affiché à 0KB.
La méthode d'effacement du cache est quelque peu différente de la méthode Android. Les dossiers et les liens symboliques sont préservés et seulement leur contenu est effacé. De cette façons lorsque vous repartez une application le lien symbolique est toujours en place et les données continueront d'aller là où nécessaire plutôt qu'entièrement dans /data/data/*.

Nandroid
Si vous choissez de faire un backup nandroid à partir de l'application vous devez savoir que cette fonction ne fonctionnera qu'avec les recovery basé sur CWM (ClockWorkMod). TWRP (Touch) n'est pas supporté pour le moment, si vous voulez aider sentez-vous libre de m'envoyer un message privé sur XDA ou par email zatta.xda@gmail.com (en anglais svp).
